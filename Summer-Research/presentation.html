TRIM

Table Retrieval and Information Minimisation

Mayank Vyas, Inkita Tewari, Niharika Agarwal

Arizona State University

Motivation

  • The challenge of information overload in vast tabular datasets.
  • Difficulty for users to pinpoint exact information from large, complex tables.
  • Existing retrieval systems often return whole tables, not just the relevant parts.
  • Need for a system that intelligently retrieves and simplifies tabular data.

Inspiration

Our work builds upon foundational research in several key areas:

Open-Domain Question Answering

Inspired by models like BERT and T5 that understand natural language queries to find answers in text.

Semantic Table Search

Building on CRAFT's cascaded retrieval approach that uses sparse embeddings for efficient table retrieval, achieving better performance than dense models while reducing computational costs. (CRAFT, 2025)

Information Theory

Applying principles of minimal information and entropy to reduce data without losing meaning.

Contrastive Learning Approach

We trained a contrastive model using one positive and two negative table approach to learn better embedding representations for table-query matching.

Dataset: NQ-Tables

About NQ-Tables

The NQ-Tables dataset is a benchmark designed for open-domain question answering systems that require information extraction from tables. It is a subset of the larger Natural Questions (NQ) dataset, which contains real user questions issued to Google Search and answers found on Wikipedia.

Key Characteristics

  • • Real User Queries: Questions issued to Google Search by actual users
  • • Wikipedia Tables: Answers derived from structured tables in Wikipedia
  • • Diverse Domains: Covers a wide range of topics and table structures
  • • Complex Reasoning: Requires understanding of table structure and content

Stage 1: Table Pruning

Row-Based Pruning

We tested 3 types of row chunks:

1. Only row data (no metadata)
2. Table title + row data ✓ Best Results
3. Table title + row data + column names

Implementation Details

Test Dataset: 919 test queries

Tables per Query: Top 50 (CRAFT SPLADE retrieval)

Embedding Model: sentence-transformers/paraphrase-multilingual-mpnet-base-v2

Process: Generate dense embeddings for row chunks and query, score each row

Key Benefits

• Reduced table size allows feeding LLM more tables

• Maintained context size while reducing noise

• Improved efficiency in downstream processing

Stage 3: Contrastive Learning

Contrastive Learning Approach

• Objective: Learn embeddings that bring relevant query-table pairs closer while pushing irrelevant pairs apart

• Architecture: Triplet loss with neural ranking model

• Training Data: Query-positive-negative triplets from CRAFT results

• Embedding Types: Both sparse (SPLADE) and dense (E5) embeddings tested

Triplet Formation Strategy

Positive Examples: Gold tables for each query

Hard Negatives: Tables ranked above gold table in CRAFT results

Selection Logic:

  • • If gold rank ≤ 3: Use top (rank+1) tables as negatives
  • • If gold rank > 3: Use top 2 tables as negatives
  • • Maximum 2 hard negatives per query

Challenges Faced

  • Data Heterogeneity: Tables come in countless formats, making robust parsing difficult.
  • Defining Relevance: Quantifying "relevance" is subjective and query-dependent, posing a challenge for model training.
  • Error Propagation: An error in an early pipeline stage (e.g., poor retrieval) significantly impacts final output quality.
  • Embedding Space Alignment: SPLADE embeddings showed poor discrimination between positive and negative examples, requiring specialized training strategies.
  • Training Stability: Basic triplet loss with hard margins led to unstable training, necessitating enhanced architectures with soft margins and advanced regularization.
